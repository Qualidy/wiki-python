    List Comprehensions¶

    List Comprehensions in Python sind eine elegante und effiziente Möglichkeit um Listen zu erstellen und Operationen auf ihren Elementen auszuführen. Sie bieten eine klare und prägnante Alternative zu traditionellen Schleifen und Funktionsaufrufen. List Comprehensions sind eine typische Struktur, die man in Python-Code häufig findet.

    Problemstellungen¶

    Video

    Stellen wir uns vor, wir möchten aus einer vorhandenen Liste eine neue Liste erstellen, in der jedes Element aufgrund einer Bedingung oder einer Operation verändert wurde. Traditionell würden wir dazu eine for-Schleife verwenden, die über die alte Liste iteriert, die Operation durchführt und das Ergebnis in einer neuen Liste speichert. Das würde dann so aussehen:

    💻 Online Compiler

    1
2
3
4
5
    quadrate = []
for i in range(1, 6):
    quadrate.append(i * i)

print(quadrate)

    List Comprehensions vereinfachen diesen Code, indem sie die gesamte Logik in eine einzige, lesbare Zeile komprimieren. Sie können Bedingungen anwenden, Funktionen aufrufen und die resultierende Liste direkt erzeugen, was den Code wesentlich sauberer und eleganter macht.

    💻 Online Compiler

    1
2
3
    quadrate = [i * i for i in range(1, 6)]

print(quadrate)

    # Without List Comprehension
quadrate = []
for i in range(1, 6):
    quadrate.append(i * i)

# List Comprehension
quadrate = [i * i for i in range(1, 6)]

    Die einfachste Form der List Comprehension sieht also so aus:

    [ausdruck(item) for item in iterable]

    Motivation für List Comprehensions¶

    • Kompakter Code: List Comprehensions ermöglichen es, Schleifen und bedingte Anweisungen in einer Zeile zu schreiben, wodurch der Code kürzer und, wenn man es nicht übertreibt, auch leichter lesbar wird.

    • Performance: Sie sind oft schneller als traditionelle Schleifen, besonders bei der Erstellung großer Listen.

    List Comprehensions sind ein hervorragendes Beispiel für pythonic Code, also Code der typisch für Python-Programme ist und die jeder kennen und verstehen sollte.

    Aufgaben zu einfachen List Comprehensions¶

    🌶 Quadrate erstellen

    Erstelle eine Liste der Quadrate der Zahlen von 1 bis 10 mit List Comprehension. Also [1, 4, 9, 16, ..., 100].

    Lösung 
    1
    quadrate = [i ** 2 for i in range(1, 11)]
    🌶 Zeichenkettenlängen

    Erstelle eine Liste mit den Längen jedes Wortes in einer vorgegebenen Liste von Wörtern.

    Lösung 
    1
    wortlaengen = [len(wort) for wort in ["Hallo", "Welt", "Python"]]
    🌶 Absolute Werte

    Wandele eine Liste von Zahlen in eine Liste ihrer absoluten Werte um. Z.B. [-1, -2, 3, -4, 5] zu [1,2,3,4,5]. Verwende dazu die Funktion abs.

    Lösung 
    1
    absolute = [abs(i) for i in [-1, -2, 3, -4, 5]]
    🌶🌶 String in Großbuchstaben

    Konvertiere jede Zeichenkette in einer Liste in Großbuchstaben.

    Lösung 
    1
    grossbuchstaben = [s.upper() for s in ["hallo", "welt", "python"]]
    🌶 Wurzeln ziehen

    Berechne die Quadratwurzel jeder Zahl in einer Liste von Zahlen.

    Lösung 
    1
    wurzeln = [i ** 0.5 for i in [1, 4, 9, 16, 25]]
    🌶 Tupel erstellen

    Erstelle eine Liste von Tupeln (x, x*x) für jede Zahl x von 1 bis 10.

    Lösung 
    1
    tupel_liste = [(i, i * i) for i in range(1, 11)]
    🌶🌶 Teile von Strings

    Erstelle eine Liste der ersten Zeichen jedes Wortes in einer Liste von Wörtern.

    Lösung 
    1
    erste_zeichen = [wort[0] for wort in ["Hallo", "Welt", "Python"]]
    🌶🌶 Durchschnittswerte

    Berechne den Durchschnitt von jedem Paar aufeinanderfolgender Zahlen in einer Liste. Also z.B. [1,2,5,5,2,-2] wird zu [1.5, 3.5, 5.0, 3.5, 0]

    Lösung 
    1
    durchschnitt = [(liste[i] + liste[i + 1]) / 2 for i in range(len(liste) - 1)]

    Elemente filtern¶

    Es ist bei einer List Comprehension einfach möglich bestimmte Elemente aus der Liste unter einer Bedingung zu entfernen. Der folgende Code:

    if len(word) > 3:
        result.append(word)

# List Comprehension
result = [word for word in words if len(word) > 3]

    Im letzten Beispiel haben wir gesehen, wie wir Elemente filtern können. Manchmal möchte man aber auch verschiedene Operationen auf einem Element durchführen, basierend auf einer Bedingung. Hier lässt sich der ternäre Operater nutzen. Den folgendne Code

    🌶 Gerade Zahlen

    Erzeuge eine Liste aller geraden Zahlen zwischen 1 und 20.

    Lösung 
    1
    gerade = [i for i in range(1, 21) if not i % 2]
    🌶 Filtern nach Bedingung

    Erzeuge eine Liste aller Zahlen von 1 bis 20, die durch 3 teilbar sind.

    Lösung 
    1
    teilbar_durch_3 = [i for i in range(1, 21) if i % 3 == 0]
    🌶 Nicht-leere Strings

    Filtere eine Liste von Strings und behalte nur die nicht-leeren bei.

    Lösung 
    1
    nicht_leer = [s for s in ["Hallo", "", "Welt", ""] if s]
    🌶🌶 Fizz Buzz

    Erstelle eine Liste von Strings "Fizz", "Buzz" oder "FizzBuzz" für Zahlen von 1 bis 15, abhängig davon, ob die Zahl durch 3, 5 oder beide teilbar ist.

    Lösung 
    1
2
    fizz_buzz = ["FizzBuzz" if i % 3 == 0 and i % 5 == 0 else "Fizz" if i % 3 == 0 else "Buzz" if i % 5 == 0 else str(i) for
             i in range(1, 16)]

        result.append(i * j)

    🌶🌶 Bedingung fehlt

    Was ist hier die Ausgabe:

    1
2
    result = [(i, j) for i in range(1, 11) for j in range(1, 11) if ...]

    Lösung 
    1
    paare = [(i, j) for i in range(1, 11) for j in range(1, 11) if i + j == 10]
    🌶🌶🌶 Liste von Listen abflachen

    Flache eine Liste von Listen zu einer einzigen Liste ab. Also z.B. aus [[1, 2], [3, 4], [5, 6]] zu [1, 2, 3, 4, 5, 6].

    Lösung 
    1
    abgeflacht = [element for sublist in [[1, 2], [3, 4], [5, 6]] for element in sublist]

    Generator - was dahintersteckt¶

    Video

    Vielleicht ist dir aufgefallen, dass die folgende Syntax nicht zu einem Tupel führt, sondern man einen Generator zurückbekommt:

    1
2
3
4
    my_gen = (i * 2 for i in range(4))

print(my_gen) # <generator object <genexpr> at 0x000001F7191D7B50>
print(type(my_gen)) # <class 'generator'>

    Tatsächlich handelt es sich hier in der ersten Zeile um eine Generator Expression.

    Ein generator kann mit der Methode next um das nächste Element gebeten werden. Das geht so lange, bis eine StopIteration Exception geworfen wird.

    1
2
3
4
5
6
7
    my_gen = (i * 2 for i in range(4))

print(next(my_gen)) # 0
print(next(my_gen)) # 2
print(next(my_gen)) # 4
print(next(my_gen)) # 6
print(next(my_gen)) # StopIteration

    So ein generator lässt sich so auch klasse mit einer for-Schleife durchlaufen, da diese intern immer wieder die next-Methode aufruft, bis es zur StopIteration Exception kommt.

    1
2
3
4
5
    summe = 0
for i in (i * 2 for i in range(4)):
    summe += i

print(summe)

    Dieser Code lässt sich tatsächlich noch kürzen zu:

    1
2
    summe = sum(i * 2 for i in range(4))
print(summe)

    Es gibt diverse eingebaute Pythonmethoden, die so mit Iteratoren umgehen können. Dazu gehören: all, any, map oder tuple. Du findest noch weiter hier in der Dokumentation, wenn du nach den Funktionen suchst, die iterable als Parameter erwarten.
