    Exception Handling in Python¶

    Video

    Beim Programmieren in Python ist es wichtig, Ausnahmen ordnungsgemäß zu behandeln, um robusten und fehlerresistenten Code zu schreiben. Ausnahmen sind Ereignisse, die während der Ausführung eines Programms auftreten und den normalen Ablauf unterbrechen.

    Du kannst durch korrektes Handling von Ausnahmen Fehler auf elegante Weise bewältigen und so die Gesamtzuverlässigkeit deines Codes verbessern.

    Grundlagen des Exception Handlings¶

    Try-Except-Block¶

    Video

    Die Blöcke try und except werden verwendet, um Ausnahmen in Python zu behandeln. Der try-Block enthält den Code, der eine Ausnahme auslösen könnte, und der except-Block gibt an, wie die Ausnahme behandelt werden soll.

    # Code, der eine Ausnahme auslösen könnte
    ergebnis = 10 / 0

    # Behandlung der spezifischen Ausnahme
    print(f"Fehler: {e}")

print("Ich lebe noch und kann weiter machen")

    pass

phone_numbers = {'Jürgen': '01234-5678', 'Monika': '+49-156-89345'}

    ⚠ Achtung: Exception Handling und Ablaufsteuerung¶

    Video

    Es ist wichtig zu betonen, dass Exception Handling nicht als Mechanismus zur Ablaufsteuerung genutzt werden sollte. Ausnahmen sollten nicht für die Kontrolle des normalen Programmflusses verwendet werden, sondern ausschließlich für die Behandlung von unerwarteten Ereignissen und Fehlerzuständen.

    🌶🌶🌶 Übertriebene Rekursion

    Zu welchem Fehler führt der folgende Code und warum?

    1
2
3
4
    def fak(n):
    return 1 if n <= 1 else n * fak(n-1)

print(fak(1000))
    Lösung

    Die Funktions fak ruft sich immer wieder selbst auf (das nennt man eine rekursive Funktion). Der Callstack wird damit immer größer, bis dieser irgendwann die maximale Größe überschreitet. Dann wird ein RekursionError geworfen.
