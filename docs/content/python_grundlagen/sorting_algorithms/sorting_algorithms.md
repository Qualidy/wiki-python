# Mini-Projekt: Sortieralgorithmen

In Mathe habt ihr ja bereits die Sortieralgorithmen Bubblesort und Mergesort kennengelernt.

Wir wollen diese nun in Python implementieren. Über die nächsten Tage sollen Sie folgende Aufgaben lösen:

### Aufgabe: Luftblasen 🌶🌶🌶
Implementieren sie [Bubblesort](https://de.wikipedia.org/wiki/Bubblesort).

### Aufgabe: Aufteilen und Herschen 🌶🌶🌶
Implementieren sie [Mergesort](https://de.wikipedia.org/wiki/Mergesort).

[Lösungen](https://medium.com/@sulthanm2d/sort-method-algorithm-in-python-359888ed8975)

### Aufgabe: Fremden Code lesen 🌶🌶🌶
[Laden Sie diese Dateien herunter🔽](sorting_example.zip). Es enthält eine Sortieralgorithmus und dessen Visualisierung.
Erklären Sie den Ablauf des Sortieralgorithmus und machen Sie sich grob mit der Visualisierung vertraut.
Schreiben Sie den Docstring für die Funktion `simple_sort`. Ggf. musst du noch `pip import matplotlib` ausführen,
damit die Visualisierung funktioniert.

### Aufgabe: Präsentation eines neuen Sortieralgorithmus 👨‍🏫👩‍🏫
Auch als Informatiker ist es wichtig, sicher vor Menschen sprechen zu können und seine Ergebnisse
ansprechend und verständlich zu präsentieren. Daher sollen sie einen Vortag zu einem Sortieralgorithmus vorbereiten.

* Wählen Sie ein Sortierverfahren aus und sprechen Sie dieses mit ihrem Dozenten ab. Hier finden sie eine Liste von [vielen weiteren Sortierverfahren](https://de.wikipedia.org/wiki/Sortierverfahren).
* Implementieren Sie das Sortierverfahren. Sie müssen in der Lage sein, ihre Implementierung vollständig vor Publikum zu coden und live anzupassen. Von Null an.
* Wenn möglich, erstellen sie eine Visualisierung für das Verfahren. Sonst nutzen sie eine Bestehende.
* Bereiten Sie sich darauf vor die Funktionalität und die Eigenschaften des Sortierverfahrens zu erklären. Achtung: Es kommt überhaupt nicht gut, wenn man Eigenschaften nennt, die man nicht erklären kann. Wer also z.B. sagt, dass der Algorithmus "stabil" ist, der muss auch erklären können, was das bedeutet.
* Überlegen Sie sich auch Randbeispiele, das heißt, die Fälle, in denen Ihr Algorithmus besonders langsam oder schnell ist.
* Überlegen Sie sich, wie sie den Sortieralgorithmus ansprechend und verständlich präsentieren können. Ziel dabei ist des, einen fachfremden Zuhörer in die Lage zu versetzen, dass er diesen Sortieralgorithmus anzuwenden (z.B. mit ein gemischten Stapel Spielkarten) 
* Erstellen Sie eine ansprechende uns aussagekräftige Markdowndatei. Diese wird dann hier ins Wiki eingepflegt.
* Üben Sie die Präsentation und präsentieren sie diese.

Ziel der Präsentation:

* Die Zuhörer sollen in der Lage sein, den Sortieralgorithmus mit einem Stapel gemischter Spielkarten vor Ort durchzuführen.
* Die Zuhörer sollen in der Lage sein den Algorithmus zu implementieren (Wenn sie aus dem Kurs sind).

Wir werden auch die Mentoren als Zuhörer einladen 😉
