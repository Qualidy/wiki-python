# Checkliste git

- Ich verstehe, wofür GIT da ist.
- Ich kann ein eigenes Repository erzeugen.
- Ich verstehe die Struktur von Git (Working Tree, Index, Repository).
- Ich kann Commits erstellen.
- Ich kann auswählen, welche Dateien und Änderungen in einen Commit mit aufgenommen werden.
- Ich kann zwischen Projektversionen navigieren.
- Ich kann Projektversionen zu einer verbinden.
- Ich kann Konflikte beim mergen Lösen.
- Ich weiß was Branches sind.
- Ich weiß, wie ich Branches verschieben kann, um z.B. Merges Rückgängig zu machen.
- Ich kann ein Repository klonen, bearbeitet und wieder hochgeladen.
- Ich kann gemeinsam mit anderen über git an einem Projekt arbeitet.
- Ich kann selbstständig nach Lösungen suchen, die bei der Nutzung von git auftreten.
